Civic Twin Builders

Start with readable files before heavy platforms. These small builders turn P4A's roots-up governance pattern into local Markdown records that humans can inspect and authorised agents can use for user-approved actions. Editing those records is a separate permissioned workflow.

Builder suite

Private first, public by consent.

The suite demonstrates the grassroots development path: L0 is self/home, L1 is local mesh, L2 is regional or civic front, and L3+ is macro scale such as state, national, international or global. The tools are fractal: a noticeboard, ledger or milestone can be private, shared or public depending on consent and scale. Most tools create local Markdown drafts that agents may use for approved actions. Changing the records is a later permissioned container, not the default right of an agent.

Public/private boundary

Markdown is the contract, not the permission.

A generated file is only a draft. Private files stay private. Permissioned files need the named people to understand the use. Public files need source trails, review status and correction paths before they become part of any ledger, noticeboard or digital twin.
